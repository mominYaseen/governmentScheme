Scheme Navigator

Discover government schemes that fit you — browse, match, save, and get reminded to apply.

Government Scheme Assistant · AI-assisted discovery

The problem

  • Thousands of Central and State schemes — hard to navigate.
  • Eligibility is buried in long PDFs and portal pages.
  • Citizens need a simple question answered: “What applies to me, and how do I apply?”

Our solution

  • Catalog — search and paginate schemes from a real dataset.
  • Structured recommend — profile → SQL-backed eligibility rules (fast, explainable).
  • AI match — describe your situation in natural language; Gemini helps extract profile and explain matches.
  • Personal list — sign in, save schemes, optional email reminders.

Feature: Scheme catalog

  • Paginated list (thousands of schemes).
  • Clear badges: Central / State — not raw import tags.
  • Categories and human-readable subtitles instead of opaque slugs.
  • “View details” for full text: overview, benefits, eligibility, documents, apply link when available.

Feature: Rule-based recommend

  • User enters structured profile: state, income, age, student/farmer flags, etc.
  • Backend uses eligibility_criteria rows (from CSV import).
  • No LLM required — good for demos and transparency.

Feature: AI match (Gemini)

  • Free-text story → structured profile + scheme ranking.
  • Legacy rule engine marks eligible vs near-miss on seed schemes.
  • Localized explanations for matched sets (with sensible fallbacks).

Requires GEMINI_API_KEY; rate limits handled with retries.

Feature: Save & remind

  • Catalog and matching stay public; saving requires login.
  • Demo email/password (or OAuth) + session-based security.
  • Saved schemes stored per user; scheduled job can send apply reminders via SMTP.

Architecture

  • Frontend — SPA (e.g. Vite + React)
  • API — Spring Boot REST
  • DB — PostgreSQL + normalized schema
  • AI — Google Gemini API
  • Import — CSV → schemes, categories, criteria
  • Docs — OpenAPI / Swagger UI

Tech stack

  • Java 21 · Spring Boot 3 · Spring Data JPA · Spring Security
  • PostgreSQL · Flyway-style SQL init (schema.sql)
  • OpenCSV dataset import · OpenAPI (springdoc)
  • Optional: OAuth2 (Google) · Spring Mail · scheduling

Data & quality

  • Large CSV (Kaggle-style) → ~3k+ scheme rows.
  • Heuristic parsing of eligibility text into typed criteria.
  • Apply URLs often missing in source data — API can infer first http(s) link from text when helpful.

API highlights

  • GET /api/schemes — paginated summaries
  • GET /api/schemes/{id} — detail + overview / apply steps
  • POST /api/schemes/recommend — structured rules
  • POST /api/schemes/match — natural language + Gemini
  • /api/me/… — profile & saved schemes (authenticated)

Demo flow (for your video)

  1. Open catalog → pick a scheme → show detail & apply section.
  2. Run recommend with a sample profile.
  3. Run match with one short story.
  4. Log in → save a scheme → open “My saved schemes”.

Future work

  • Production auth, HTTPS, and hardened CORS.
  • Regional languages and accessibility.
  • Admin tools for curating schemes and URLs.
